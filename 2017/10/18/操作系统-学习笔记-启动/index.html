操作系统-学习笔记-启动

BIOS：

将Bootloader从磁盘引导扇区(512bytes)加载到0x7c00
跳转到CS：IP=0000：7c00

Bootloader

将操作系统代码和数据从硬盘加载到内存中

系统调用：应用程序主动向操作系统发出

异常(来源于不良的应用程序)：非法指令或其他坏的处理状态(内存出错)

中断(来源于设备)：来自不同的硬件设备的计时器和网络中断

处理时间：
中断：异步
异常：同步
系统调用： 同步或异步

响应时间：
中断：持续，对用户应用程序是透明的
异常：杀死或重新执行意想不到的应用程序指令
系统调用：等待和持续

中断 中断表

硬件

设置中断标记[cpu初始化]

  1. 将内部、外部事件设置中断标记
  2. 中断事件的id
软件
  1. 保存当前处理的状态
  2. 中断服务程序处理
  3. 清楚中断标记
  4. 恢复之前保存的处理状态

异常：异常编号

保存现场
异常处理
  1. 杀死产生了异常的程序
  2. 重新执行异常指令
    恢复现场(也是透明的，用户感知不到)

系统调用

系统调用接口
例： 应用程序调用printf() ，会 调用系统调用write()

接口而不是直接进行系统调用

win32 api：windows
posix api：posix-base systems，包括unix，linux，mac
java api：jvm

操作系统 用户态，内核态

跨越操作系统边界的开销

  • 在执行时间上的开销超过程序调用
  • 开销：
    建立中断/异常/系统调用号与对应服务例程映射关系的初始化开销
    建立内核堆栈
    验证参数
    内核态映射到用户态的地址空间，更新页面映射权限
    内核态独立地址空间 TLB
